## 3. マイルストーン1 機能要件

### 3-1. タグ機能

**目的:** Item を分類・フィルタリングし、関連する思考を素早く発見できるようにする。

**スコープ:**

- Item にタグを複数付与できる
- タグでItemを絞り込み表示できる
- タグは自由入力で作成できる（事前定義不要）

**推奨データモデル:**

```
tags テーブル
  id    : bigint PK
  name  : string NOT NULL, UNIQUE

item_tags テーブル（中間テーブル）
  item_id : bigint NOT NULL, FK → items.id
  tag_id  : bigint NOT NULL, FK → tags.id
  ユニーク制約: (item_id, tag_id)
```

**モデル関係:**

- `Item has_many :item_tags → has_many :tags, through: :item_tags`
- `Tag has_many :item_tags → has_many :items, through: :item_tags`

**gem 方針:** `acts-as-taggable-on` は不使用。ゼロから実装（理由: オーバースペック・Rails 7.2 互換リスク）

---

### 3-2. デイリーノート機能

**目的:** 毎日開く習慣をつくる。その日の思考・出来事・気づきをキャプチャする定点記録。

**スコープ:**

- 特定の日付に紐づく 1 件のノートを作成・表示できる
- 今日のデイリーノートにすぐアクセスできる
- 過去のデイリーノートを一覧で遡れる

**推奨データモデル（Item 拡張案）:**

```
items テーブルへ追加:
  item_type : string NOT NULL, DEFAULT 'note'  ("note" or "daily")
  note_date : date   NULL可（daily の場合のみ使用）
```

**理由:** 別テーブルにするとタグ機能でポリモーフィック設計が必要になり複雑度が増す。Item を拡張する方が変更コストが小さい。
